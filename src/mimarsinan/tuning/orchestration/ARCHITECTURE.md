# tuning/orchestration/ -- Smooth-adaptation orchestration loop + services

The control core of the tuning subsystem: the rate-search loop, the per-cycle
predictor→corrector→commit/rollback machinery, the adaptation-manager rate host,
and the decision services the refactor is extracting from the loop.

## Key Components

| File | Symbols | Purpose |
|------|---------|---------|
| `tuner_base.py` | `TunerBase`, `CATASTROPHIC_DROP_FACTOR`, `_RECOVERY_PATIENCE`, `_STUCK_STREAK_REQUIRED` | Shared infrastructure: trainer, budget, target adjuster, LR finder (`_find_lr`/`_get_cached_lr` with `anchor_lr=pipeline_lr`), `validate`. Tuning constants. `_create_trainer` builds the `BasicTrainer` with `recipe=self._tuning_recipe()` (the `tuning_recipe` block) and forwards `tuning_recipe_recovery` (config flag, default off → byte-identical) so the STEP recovery honors the recipe optimizer + warmup/cosine schedule instead of the hardcoded Adam/constant-LR path. |
| `smooth_adaptation_cycle.py` | `SmoothAdaptationCycleMixin` | One adaptation cycle: clone pre-state → pre-cycle eval → `_update_and_evaluate(rate)` probe → catastrophic gate → LR → recovery train → dual rollback gate → commit + stuck-streak target relaxation. Records one `DecisionRecord` per exit into `self._cycle_log`. Accept/reject decisions delegate to `AcceptanceSensor` (static methods); `_absolute_post_acc_floor` delegates too. **Recovery-quality knobs** (all default-off → byte-identical): (1) `tuning_refind_lr_on_miss` invalidates the cached LR when a committed cycle misses the target so the next cycle re-discovers it (else the LR is cached at cycle 0 forever); (2) `tuning_recovery_lr_plateau` (+`_factor`/`_reductions`) passes plateau LR-reduction kwargs to `RecoveryEngine.train_to_target` via `_recovery_plateau_kwargs` (off → 1.0/0 no-op); (3) `tuning_rollback_ratchet` (+`tuning_rollback_cumulative_bound`, default 0.05) is the NON-STALLING ratchet: `AcceptanceSensor.ratchet_threshold` keeps the per-step relative gate (`pre_cycle_acc - margin`, so the ramp keeps climbing) but caps the cumulative drift below the `_best_committed_acc` high-water mark (`_ratchet_best_anchor`, tightened on each commit by `_update_best_committed_acc`) at the bound — repeated sub-margin slips cannot accumulate and the bound tightens as best ratchets up; (4) `tuning_tight_plateau` (+`tuning_recovery_check_divisor`) divides the recovery check interval (`_recovery_check_interval`) so the stale-streak patience trips after fewer steps (stacks with the plateau LR reduction); (5) `tuning_stabilization_bounded` makes the gradual recovery call request `return_steps` and `_accumulate_gradual_steps` sums them into `_gradual_train_steps` (the bounded-stabilization budget, consumed in `smooth_adaptation_run.py`). **GENUINE full-transform probe** (`tuning_full_transform_probe`, default off): after each commit, `_probe_full_transform` measures BOTH the value-domain (`_value_full_transform_eval`, always non-destructive clone→rate-1.0→restore) AND the genuine deployed-forward (`_full_transform_eval`, base default = value; overridden by `KDBlendAdaptationTuner`) rate-1.0 accuracy. Logs a per-commit record `{committed, committed_acc, value_full_acc, genuine_full_acc, value_drop, genuine_drop, proxy_gap}` (with `full_acc`/`drop` aliasing the value pair for the legacy test). `proxy_gap = value_full_acc − genuine_full_acc` is the value↔genuine divergence the value proxy hides. The verdict (`_log_full_transform_trend`, in `smooth_adaptation_run.py`) keys on `genuine_drop`. |
| `smooth_adaptation_run.py` | `SmoothAdaptationRunMixin` | `run()`: baseline calibration via `AcceptanceSensor.calibrate_baseline` → `_run_with_scheduler` (the single `RateScheduler` loop, attempt = `_adaptation`) → `_finalize_run` (`_after_run` → `_stabilize_at_full_rate`). `_continue_to_full_rate` is retained as an `_after_run` helper for the rate-tuner families. **Bounded cosine stabilization** (`tuning_stabilization_bounded`, +`tuning_stabilization_ratio` default 0.5): when on, `_stabilize_at_full_rate` delegates to `_stabilize_bounded_cosine` — a SINGLE hard-cutoff pass of `int(ratio * _gradual_train_steps)` steps with a CosineAnnealingLR decay (chosen LR → ~0 over exactly N steps, via `train_steps_until_target(cosine_decay=True)`, unreachable target + `check_interval=N` so no early break), bracketed by the same pre/post rollback guard (non-destructive). Off → the existing patience/round-based pass (`_stabilization_budget` × `_max_stabilization_rounds`, cached LR) unchanged. `_log_cycle_summary` prints the `DecisionTrace` and (if the probe ran) the trend. **`_log_full_transform_trend`**: verdict keys on the GENUINE drop — CONVERGING when `last.genuine_drop < first.genuine_drop` (deployed cliff shrinking), else FLAT/DIVERGING. When genuine fails to shrink but `value_drop` DID, it warns "the value-domain probe WAS FOOLED" and surfaces the `proxy_gap` trajectory; the value drop alone is no longer a convergence signal. |
| `smooth_adaptation_tuner.py` | `SmoothAdaptationTuner` (= Cycle+Run+Base mixins), `UnifiedPerceptronTuner` (alias) | The composed orchestration tuner all rate tuners extend. |
| `acceptance_sensor.py` | `AcceptanceSensor`, `BaselineRef` | The accept/reject/recovered decision service. Pure, bit-exact extraction of the loop's statistical math: `calibrate_baseline` (SE, empirical noise, clamped rollback tolerance, fixed baseline), `absolute_floor` (cumulative-drift guard), `is_catastrophic`/`rollback_threshold`/`ratchet_threshold`/`is_rollback`/`reached_target`. `ratchet_threshold(pre_cycle_acc, margin, best, cumulative_bound, floor)` is the NON-STALLING ratchet (`tuning_rollback_ratchet`): the MAX of the per-step relative bound (`pre_cycle_acc - margin`, so the ramp keeps climbing — no best-anchor stall), the cumulative-drift bound (`best - cumulative_bound`, caps total give-back below the best high-water mark, tightens as best ratchets up), and the absolute floor. **Paired McNemar gate (P2b)**: `paired_drop_se` / `paired_is_rollback` test the candidate vs the fixed-baseline correctness vector on a shared subsample (`(b10-b01)/N`, SE `sqrt(b10+b01)/N` — several-fold tighter than the marginal `0.5/sqrt(n)`). Wired into the rollback gate behind `tuning_use_paired_sensor` (off → marginal path bit-exact); Monte-Carlo-calibrated in `test_paired_sensor_calibration`. |
| `rate_scheduler.py` | `RateScheduler` | The single rate-search policy (spec §5.2 greedy-to-1.0 then bisect-the-gap) that replaced the legacy one-shot + grow/halve ramp + `_continue_to_full_rate` loops. The first (greedy/ladder) jump always runs; `epsilon` bounds only the bisection. Policies: `greedy_to_one` (default), `uniform_ladder` (KD-blend), `one_shot_only`. |
| `adaptation_driver.py` | `AdaptationDriver` | Thin orchestrator: `run()` drives the scheduler from `committed` toward 1.0 over a per-cycle `attempt` (predictor→corrector→commit/rollback), then `finalize`. `build_scheduler` selects the policy. The sole run loop, via `SmoothAdaptationRunMixin._run_with_scheduler` (attempt = the tuner's `_adaptation`, finalize = `_finalize_run`). The fully standalone form (built from model+axis+services, dissolving `_adaptation` into `_cycle`) is the remaining V6 step. |
| `characterization.py` | `characterize`, `Profile` | Pre-search profiling (spec §10 / V9): sweeps the paired drop on an α grid → `Profile(monotonic, max_slope, epsilon_hint, feasible_max, drops)`. A non-monotonicity beyond the noise budget (A1) is the trigger for dense-grid safe mode; the steepest slope sets `epsilon_hint` (A3) so bisection never steps over a cliff. |
| `recovery_engine.py` | `RecoveryEngine`, `PersistentOptimizerOwner`, `RESET_PER_CYCLE`, `PERSIST_WITHIN_CYCLE` | The corrector. `train_to_target` centralizes the `train_steps_until_target` + recovery-hook remove-in-`finally` pattern (per-cycle recovery, below-floor safety net, stabilization) and threads an optional owned `optimizer`. **Optimizer policy**: the cycle's `_optimizer_policy()`/`_recovery_optimizer(lr)` pass `optimizer=None` by default (fresh build + del each call — bit-exact), or, under the `tuning_persist_optimizer` opt-in, a `PersistentOptimizerOwner`-built optimizer reused across recovery calls so Adam moments survive (owner reset at `run()` start, getattr-guarded). Tuner families whose recovery replaces the parameter set each cycle set `_supports_persistent_optimizer = False` and always reset. LR discovery / tunable-param folding remain later extensions. |
| `checkpoint_guard.py` | `CheckpointGuard`, `Handle` | Scoped, location-aware rollback snapshots. `scope=full`/`location=device` (default) delegates verbatim to `clone/restore_state_for_trainer` (byte-identical, golden-safe); `scope=tunable` clones only `requires_grad` params + buffers (skip frozen backbone); `location=cpu_pinned` offloads to CPU (frees ~1× model VRAM, syncs before restore). The sole snapshot path in `_clone_state`/`_restore_state`; `scope`/`location` are config opt-ins defaulting to full/device. |
| `adaptation_manager.py` | `AdaptationManager` | Rate-field host (`clamp_rate`/`quantization_rate`/`shift_rate`/`noise_rate`/`activation_adaptation_rate`/`scale_rate`); `update_activation` rebuilds a perceptron's decorator stack from current rates. |
| `adaptation_manager_factory.py` | `create_adaptation_manager_for_model` | Build a manager + run initial `update_activation` for all perceptrons. |
| `tuning_budget.py` | `TuningBudget`, `tuning_budget_from_pipeline`, `max_total_training_steps`, `min_step_for_smooth_adaptation` | Step-based budget from dataset size; `accuracy_se()` = `0.5/sqrt(eval_sample_count)`. |
| `genuine_probe.py` | `eval_forward_over_val`, `genuine_acc_on_clone` | Pure helpers for the genuine full-transform probe. `eval_forward_over_val(trainer, forward_obj, model, n_batches, device)` = top-1 accuracy of an arbitrary `forward_obj` over `n_batches` of `trainer.iter_validation_batches` (never installs it on the model). `genuine_acc_on_clone(model, device, *, prepare, build_forward, evaluate)` runs the genuine eval on a `deepcopy` (prepare → build forward → evaluate), leaving the live model's state, activation structure, and installed `forward` untouched. Both are callable-driven (no tuner imports) so they stay testable in isolation. |
| `kd_blend_adaptation_tuner.py` | `KDBlendAdaptationTuner`, `BlendActivation`, `_InstalledForward` | ANN→SNN blend ramp (LIF/TTFS base): live `BlendActivation.rate` via `set_blend_rate`, KD recovery against a frozen teacher, parity-critical `_finalize` forward-install. **Shared genuine-forward builder `_finalize_forward_for(model)`** (default `None`; tuners override — ttfs cascaded → `_SegmentSpikeForward(model, T)`, cycle-accurate LIF → its NF forward): `_finalize_forward` and the finalize install both route through it, so the probed forward ≡ the deployed forward. **`_finalize_rebuild(model)`** is the model-parameterized finalize-style activation rebuild (`_before_/_after_finalize_rebuild` hooks default to `self.model`), shared by `_finalize` (live model) and the probe (isolated clone) so probe ≡ deploy. The overridden **`_full_transform_eval`** runs that deployed forward at rate 1.0 on a deepcopy via `genuine_acc_on_clone` (clone → `set_blend_rate(1)` → `_finalize_rebuild` → build forward → `eval_forward_over_val`), snapshotting/restoring `_finalize_manager_flags` so the shared `adaptation_manager` survives; it falls back to the value-domain measure when the builder is `None`. **`_make_axis()`** picks the rate axis (default `BlendAxis`; ttfs annealed-ramp → `TTFSGenuineAxis`). `_after_run` records `finalize_cliff = ramp_metric − post_finalize` and `_report_cliff_probe_consistency` warns if the last per-commit `genuine_drop` and `finalize_cliff` (same gap, different anchor) diverge > 0.1. |
